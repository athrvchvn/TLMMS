\section{Firebase Services Used}

MMS V2 uses four Firebase services:
\begin{itemize}[leftmargin=1.5em]
    \item \textbf{Firestore}: Permanent document database. Users, machines, sessions, revoked cards, admins.
    \item \textbf{Realtime Database (RTDB)}: Live state. Machine states, pending commands, OTA metadata, bridge heartbeat.
    \item \textbf{Storage}: Binary file hosting. Firmware \texttt{.bin} files for OTA.
    \item \textbf{Hosting}: Web app CDN hosting. Serves the React admin dashboard.
    \item \textbf{Authentication}: Admin user login. Email/password provider.
\end{itemize}

\section{Why Two Databases?}

Firestore and RTDB serve different purposes:

\begin{table}[H]
\centering
\caption{Firestore vs RTDB usage in MMS V2}
\begin{tabular}{L{4.5cm}L{5.5cm}L{4.5cm}}
\toprule
\textbf{Aspect} & \textbf{Firestore} & \textbf{RTDB} \\
\midrule
Data model & Documents with subcollections & Flat JSON tree \\
Primary use & Historical records & Live state / commands \\
Write pattern & Write once, read rarely & Write frequently, overwrite \\
Pricing & Per read/write/delete & Per MB downloaded \\
Offline client support & Yes (Firebase SDK) & Yes (Firebase SDK) \\
Suitable for & Sessions, users, machines, revoked & Node status, commands, OTA \\
\bottomrule
\end{tabular}
\end{table}

\section{Firestore Schema}

\subsection{/users/\{roll\_number\}}

One document per student. Document ID is the 9-digit roll number.

\begin{lstlisting}[style=jsonlstyle,caption={/users/220002123}]
{
  "roll_number":          "220002123",
  "name":                 "Student Name",
  "email":                "student@iiti.ac.in",
  "pin_hash":             "aabbccdd11223344",
  "card_uid":             "A1B2C3D4",
  "machine_permissions":  5,
  "issued_at":            "2024-06-01T10:00:00Z",
  "issued_by":            "admin_uid",
  "is_active":            true,
  "last_seen":            "2024-06-15T14:30:00Z"
}
\end{lstlisting}

\subsection{/machines/\{machine\_id\}}

One document per machine. Document ID is the string representation of the machine ID (``1'', ``2'', etc.).

\begin{lstlisting}[style=jsonlstyle,caption={/machines/3}]
{
  "name":                  "Soldering Station 1",
  "session_limit_minutes": 15,
  "relay_active_high":     true,
  "machine_active":        true,
  "location":              "Zone C",
  "created_at":            "2024-06-01T00:00:00Z"
}
\end{lstlisting}

\subsection{/sessions/\{auto\_id\}}

One document per completed session. Document ID is auto-generated by Firestore.

\begin{lstlisting}[style=jsonlstyle,caption={/sessions/auto-generated-id}]
{
  "machine_id":   "3",
  "machine_name": "Soldering Station 1",
  "roll_number":  "220002123",
  "user_name":    "Student Name",
  "start_time":   "2024-06-15T10:00:00Z",
  "end_time":     "2024-06-15T10:15:00Z",
  "duration_s":   900,
  "end_reason":   "card_removed",
  "status":       "completed"
}
\end{lstlisting}

Valid \texttt{end\_reason} values: \texttt{card\_removed}, \texttt{timeout}, \texttt{remote\_stop}, \texttt{power\_loss}, \texttt{watchdog}.

\subsection{/revoked/\{card\_uid\}}

Document ID is the card UID hex string (e.g., ``A1B2C3D4'').

\begin{lstlisting}[style=jsonlstyle,caption={/revoked/A1B2C3D4}]
{
  "uid":         "A1B2C3D4",
  "roll_number": "220002123",
  "revoked_at":  "2024-06-15T12:00:00Z",
  "revoked_by":  "admin_uid",
  "reason":      "lost"
}
\end{lstlisting}

\subsection{/admins/\{uid\} and /super\_admins/\{uid\}}

One document per admin user. Document ID is the Firebase Auth UID.

\begin{lstlisting}[style=jsonlstyle,caption={/admins/firebase-auth-uid}]
{
  "email":      "admin@iiti.ac.in",
  "name":       "Admin Name",
  "created_at": "2024-06-01T00:00:00Z",
  "is_active":  true
}
\end{lstlisting}

\section{Firestore Security Rules}

\begin{lstlisting}[caption={v2/firestore.rules}]
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Sessions: read by admins, write NEVER from client (bridge uses Admin SDK)
    match /sessions/{sessionId} {
      allow read: if request.auth != null &&
        (exists(/databases/$(database)/documents/admins/$(request.auth.uid)) ||
         exists(/databases/$(database)/documents/super_admins/$(request.auth.uid)));
      allow write: if false;
    }

    // Users: admins can read and write
    match /users/{rollNumber} {
      allow read, write: if request.auth != null &&
        (exists(/databases/$(database)/documents/admins/$(request.auth.uid)) ||
         exists(/databases/$(database)/documents/super_admins/$(request.auth.uid)));
    }

    // Machines: admins can read and write
    match /machines/{machineId} {
      allow read, write: if request.auth != null &&
        (exists(/databases/$(database)/documents/admins/$(request.auth.uid)) ||
         exists(/databases/$(database)/documents/super_admins/$(request.auth.uid)));
    }

    // Revoked: admins can read and write
    match /revoked/{uid} {
      allow read, write: if request.auth != null &&
        (exists(/databases/$(database)/documents/admins/$(request.auth.uid)) ||
         exists(/databases/$(database)/documents/super_admins/$(request.auth.uid)));
    }

    // Admins: any authenticated user can read (for login check)
    match /admins/{uid} {
      allow read: if request.auth != null;
      allow write: if request.auth != null &&
        exists(/databases/$(database)/documents/super_admins/$(request.auth.uid));
    }

    // Super admins: super admin writes only
    match /super_admins/{uid} {
      allow read: if request.auth != null;
      allow write: if request.auth != null &&
        exists(/databases/$(database)/documents/super_admins/$(request.auth.uid));
    }
  }
}
\end{lstlisting}

\section{Firestore Indexes}

Three composite indexes are required for efficient queries on the sessions collection:

\begin{lstlisting}[style=jsonlstyle,caption={v2/firestore.indexes.json}]
{
  "indexes": [
    {
      "collectionGroup": "sessions",
      "queryScope": "COLLECTION",
      "fields": [
        {"fieldPath": "machine_id", "order": "ASCENDING"},
        {"fieldPath": "start_time", "order": "DESCENDING"}
      ]
    },
    {
      "collectionGroup": "sessions",
      "queryScope": "COLLECTION",
      "fields": [
        {"fieldPath": "roll_number", "order": "ASCENDING"},
        {"fieldPath": "start_time", "order": "DESCENDING"}
      ]
    },
    {
      "collectionGroup": "sessions",
      "queryScope": "COLLECTION",
      "fields": [
        {"fieldPath": "status", "order": "ASCENDING"},
        {"fieldPath": "start_time", "order": "ASCENDING"}
      ]
    }
  ]
}
\end{lstlisting}

Indexes must be deployed before the bridge can run session deduplication queries. Deploy with \texttt{firebase deploy --only firestore:indexes}. Building takes 5--10 minutes.
