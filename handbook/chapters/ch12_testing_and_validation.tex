\section{Validation Strategy}

MMS V2 validation is done at three levels:

\begin{enumerate}[leftmargin=1.5em]
    \item \textbf{Hardware validation}: Each component on a node is tested in isolation with a dedicated Arduino sketch before production firmware is flashed.
    \item \textbf{Integration testing}: All system components are tested together end-to-end after provisioning.
    \item \textbf{Production verification}: After deploying to a live machine, basic access tests are repeated to confirm real-world behaviour.
\end{enumerate}

\section{Integration Test Suite}

Integration tests verify the entire system working together. They are documented in the Deployment Package (Section 12) and briefly summarised here:

\begin{table}[H]
\centering
\caption{Integration Test Summary}
\begin{tabular}{C{1cm}L{4.5cm}L{9cm}}
\toprule
\textbf{\#} & \textbf{Test} & \textbf{Verifies} \\
\midrule
1 & Normal session & Card tap $\rightarrow$ relay ON $\rightarrow$ RTDB updates $\rightarrow$ card remove $\rightarrow$ Firestore session logged \\
2 & Access denial & Non-permitted card $\rightarrow$ relay stays OFF $\rightarrow$ no Firestore document \\
3 & Session timeout & Card in slot past limit $\rightarrow$ relay turns OFF automatically $\rightarrow$ end\_reason: timeout \\
4 & Remote stop & Dashboard Stop button $\rightarrow$ relay OFF within 2s $\rightarrow$ end\_reason: remote\_stop \\
5 & Card revocation & Revoke in web app $\rightarrow$ node denies within 5s \\
6 & Offline session & RPi down $\rightarrow$ access still granted $\rightarrow$ session in LittleFS $\rightarrow$ reconnect $\rightarrow$ session in Firestore \\
7 & Stale command TTL & Command posted when offline $\rightarrow$ reconnect after 6 min $\rightarrow$ command discarded \\
8 & Config push & Change session limit in web app $\rightarrow$ node applies within 5s \\
\bottomrule
\end{tabular}
\end{table}

A node passes integration testing if Tests 1--8 all pass.

\section{Regression Testing After Firmware Updates}

After any firmware update (OTA or direct flash), run at minimum:
\begin{itemize}[leftmargin=1.5em]
    \item Integration Test 1 (normal session)
    \item Integration Test 4 (remote stop)
    \item Integration Test 5 (revocation)
\end{itemize}

This takes approximately 15 minutes and confirms the core access control and command paths still work.
