\section{Architecture Overview}

MMS V2 uses a three-tier architecture:

\begin{enumerate}[leftmargin=1.5em]
    \item \textbf{Edge tier}: ESP32 access nodes. One per machine. Physical access control, session timing, display.
    \item \textbf{Local bridge tier}: Raspberry Pi running Mosquitto (MQTT broker) and \texttt{bridge.py}. Bridges the local MQTT network and Firebase.
    \item \textbf{Cloud tier}: Firebase (Firestore + RTDB + Storage + Hosting + Auth). Permanent storage, real-time state, web hosting.
\end{enumerate}

\section{Communication Paths}

\subsection{Access Decision (Offline-capable)}

The access decision path requires no network:
\begin{enumerate}[leftmargin=1.5em]
    \item HC89 slot sensor detects card insertion
    \item MFRC522 reads card UID
    \item ESP32 derives sector key: $\text{HMAC-SHA256}(\text{master\_key},\, \text{uid\_hex})[0..5]$
    \item ESP32 authenticates Sector 1, reads Blocks 4 and 5
    \item ESP32 checks UID against LittleFS revocation list
    \item ESP32 checks permission bitmask: $(perm\_mask \gg (machine\_id - 1)) \& 1$
    \item If all checks pass: relay ON, session starts
\end{enumerate}

\subsection{Session Event Flow}

\begin{enumerate}[leftmargin=1.5em]
    \item Session ends (card removed, timeout, or remote stop)
    \item ESP32 builds JSON event: \texttt{\{ts, roll, machine\_id, event, duration\_s, end\_reason\}}
    \item If MQTT connected: publish to \texttt{mms/nodes/\{id\}/event} (QoS 1)
    \item If MQTT disconnected: append to LittleFS \texttt{/logs/sessions.jsonl}; publish on next connect
    \item bridge.py receives event via MQTT subscription
    \item bridge.py checks Firestore for duplicate (±5s deduplication window)
    \item bridge.py writes session to Firestore \texttt{/sessions} collection
    \item bridge.py updates \texttt{/users/\{roll\}/last\_seen}
\end{enumerate}

\subsection{Remote Stop Flow}

\begin{enumerate}[leftmargin=1.5em]
    \item Admin clicks Stop in web dashboard
    \item Web app writes to RTDB \texttt{/mms/commands/\{id\}}: \texttt{\{command:"stop", timestamp:now\}}
    \item bridge.py \texttt{onValue} listener triggers
    \item bridge.py checks command age: if $>$300s, discard and acknowledge
    \item bridge.py publishes to \texttt{mms/nodes/\{id\}/command} (QoS 1)
    \item ESP32 MQTT callback receives command
    \item ESP32 checks command timestamp: if $>$300s old, discard
    \item ESP32 turns relay OFF, logs end\_reason: \texttt{remote\_stop}
    \item ESP32 publishes session event; acknowledges command to RTDB
\end{enumerate}

\subsection{Revocation Flow}

\begin{enumerate}[leftmargin=1.5em]
    \item Admin marks card as lost in web app
    \item Web app creates \texttt{/revoked/\{uid\}} in Firestore
    \item bridge.py Firestore \texttt{onSnapshot} triggers
    \item bridge.py rebuilds revocation list from all \texttt{/revoked} documents
    \item bridge.py publishes \texttt{mms/all/revoked} (retained, QoS 1): \texttt{\{uids:[...], updated\_at:ts\}}
    \item All connected nodes receive the update; update LittleFS \texttt{/config/revoked.json}
    \item On next card tap: UID checked against revocation list $\rightarrow$ denied
    \item Offline nodes: receive update on reconnect (retained message delivered automatically)
\end{enumerate}

\subsection{OTA Update Flow}

\begin{enumerate}[leftmargin=1.5em]
    \item Super admin uploads \texttt{.bin} to Firebase Storage; copies download URL
    \item Super admin writes to RTDB \texttt{/mms/ota}: \texttt{\{firmware\_url, target\_version, released\_at\}}
    \item bridge.py RTDB OTA listener triggers
    \item bridge.py publishes OTA command to all nodes: \texttt{mms/nodes/+/command}
    \item ESP32 receives command; transitions to OTA state; relay OFF
    \item ESP32 calls \texttt{esp\_https\_ota()} with firmware URL
    \item On success: reboot to new partition; update NVS fw\_version
    \item On failure: if 3 crashes on new partition $\rightarrow$ rollback to previous partition
\end{enumerate}

\begin{notebox}
\textbf{Last Will and Testament (LWT):} Each ESP32 registers an LWT on \texttt{mms/nodes/\{id\}/status} with payload \texttt{\{"state":"offline"\}} (retained). If the node disconnects unexpectedly, the broker publishes this automatically. The bridge detects it and updates the RTDB node state to \texttt{"offline"}, which the dashboard reflects immediately.
\end{notebox}
